# Revision Playbook

## Phase B Ergänzungen

- autoritativer lokaler Demo-HTTP-Server
- in-memory Vertical Slice für Karte, Aktionen, Markt und Journal
- public-safe JSON-API für Terminal- und Web-Client

## Bei künftigen Fixes

1. konkrete Logs oder Repro anfordern
2. Ursache minimalinvasiv beheben
3. Versionsstand **nicht erhöhen**, solange es ein reiner Fix bleibt
4. Client, WWW und Wiki auf Auswirkungen mitprüfen
